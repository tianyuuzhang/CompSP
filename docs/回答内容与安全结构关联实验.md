# 回答内容、攻击指标与安全结构的三方关联实验

更新时间：2026-07-11

## 研究问题

本实验区分三个变量层次：

1. 指令攻击性：由同一指令多次访问得到的 ASR、ALR 表示。
2. 模型内部安全结构：白盒中使用最后层 hidden state 学习的 Ridge 伪安全投影；黑盒中只能把 ASR/ALR 诱导的稳定低维行为坐标作为“隐安全结构”。
3. 模型输出内容：单条或少量回答中可观测的长度、拒绝形式、词法语义和回答表示。

现阶段目标是验证三者存在跨问题泛化的统计关联。仅凭相关性不能证明内部安全结构因果地生成某类回答；因果结论需要后续激活干预实验。

## 默认口径

- “长内容”优先指去除开头拒绝套话后的实质内容；完整长度保留为朴素基线。
- 分别评估仅回答、仅 q1 和 q1+回答。当前脚本首先实现“仅回答”。
- `toxic`、`toxicA` 和安全判定文本禁止作为输入特征，避免标签泄漏。
- 少量回答使用 `n=1,2,4,8,20`，观察嗅探性能随访问次数的变化。
- 训练/测试继续使用固定 60/20 问题划分，禁止随机按指令拆分。

## 由浅入深的分析顺序

### 第一层：可解释行为特征

提取字符/词数、实质内容长度、拒绝开头、拒绝词命中、列表、代码块、词汇多样性等特征。对 ASR、ALR 和白盒 `pseudo_score` 分别做 Ridge 回归，并报告 Pearson、Spearman、MAE、R2 和同题序关系准确率。

这一层用于回答“长度或拒绝模板是否已经解释主要信号”。若只在不同 ALR 粗桶之间有效，不能作为安全结构证据。

### 第二层：词法内容

删除常见拒绝前缀后，对实质文本使用 word/character TF-IDF。它比纯长度更接近“长回答具体说了什么”，同时仍可通过高权重词组审计捷径。

### 第三层：语义表示

后续将回答编码为 frozen embedding，比较线性 Ridge、PLS 和 CCA。严格使用训练问题拟合降维与回归参数，在测试问题上评估。

### 第四层：白盒回答轨迹

在 Llama 回答 token 上分别提取 assistant 起始、拒绝前缀结束、实质内容中段和末 token hidden state，检验这些位置的表示与输入 q1 安全投影、ASR/ALR 的关系。

### 第五层：少样本嗅探与干预

用单条或少量回答反推 q1 的伪安全投影，绘制 `n=1/2/4/8/20` 学习曲线；随后沿安全方向干预激活，检验回答内容特征是否随投影发生可重复变化。

## 代码入口

第一、二层共用：

```bash
PYTHONPATH=$PWD/src /remote-home/zty/conda/LM_zty/repetition/bin/python \
  scripts/analyze_response_safety_structure.py \
  --scores outputs/pseudo_safety_direction/mixed_alr_last_layer_scores.jsonl \
  --sample-sizes 1,2,4,8,20 \
  --methods handcrafted,tfidf \
  --output-dir outputs/response_safety_structure/mixed_alr_last_layer
```

建议先用 `--methods handcrafted` 跑全量轻量基线，再用 `--methods tfidf --sample-sizes 1,2,4` 控制内存。预计手工特征全量需要 10~40 分钟，TF-IDF 约 1~3 小时，实际以脚本逐个 `sample_size` 的进度输出为准。

## 必须报告的负对照

- 仅长度和拒绝词模型。
- 在每个问题内打乱 `pseudo_score`。
- 随机方向投影。
- 去掉拒绝开头及安全关键词后的文本模型。
- ALR 两端都位于 `(0,1)` 的子集。
- 按攻击方案分别报告 OFA、PAIR、DrAttack，避免攻击模板差异冒充安全结构。
