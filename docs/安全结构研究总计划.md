# 安全结构研究总计划

更新时间：2026-07-07

## 1. 当前研究判断

前期 CompSP 的贡献是证明黑盒安全行为存在可学习的排序信号。现在更值得推进的问题不是继续强调“预测 ASR 很准”，而是解释这个排序信号为什么能跨主题泛化、为什么会系统性排错，以及它是否对应白盒模型中已经观察到的低维安全结构。

目前最有价值的观察是：CompSP 的两两准确率较高，但由两两比较推成全局排序后会出现明显错排。这不一定是坏消息。它可能说明 CompSP 学到的不是经验 ASR 本身，而是某种更稳定、更结构化的安全行为坐标。经验 ASR 是采样、判定、模型随机性和攻击格式共同作用的结果；白盒安全方向也不必精确等同 ASR。如果 CompSP 和白盒结构在相对于 ASR 的偏差上有一致性，就可以支持“黑盒交互泄露了模型内部安全结构的可学习投影”这一更强叙事。

## 2. 核心研究问题

主问题：

> 黑盒代理模型 CompSP 学到的排序信号，是否与白盒模型中的拒绝方向、有害性方向或多维安全子空间存在可测量的一致性？

拆成三个可检验问题：

1. 表征存在性：Llama-3.1-8B-Instruct 上能否复现稳定的拒绝/安全方向，并预测 ALR 或 ASR？
2. 黑白盒对齐：CompSP BTL 分数与白盒方向/子空间分数是否相关，尤其是在控制经验 ASR/ALR 后仍然相关？
3. 误差结构：当 CompSP 相对于经验 ASR/ALR 排错时，白盒方向是否更倾向于与 CompSP 同向，而不是随机同向？

## 3. 假设分层

### H1：低维安全信号存在

Llama 的安全行为可以由少数方向或子空间显著预测。第一层证据是白盒方向对 ALR 的两两准确率高于随机和词法 baseline。

### H2：CompSP 学到的不是纯 ASR 噪声

CompSP BTL 与真实 ASR/ALR 的关系不完美，但它与白盒方向分数、回答长度代理或 prompt 结构维度存在稳定相关。

### H3：错排有结构

在同题内 pair 上，如果 `CompSP_label != empirical_label`，白盒标签与 CompSP 标签的一致率高于置换 baseline。这个假设最关键，但也最容易被方向符号、层选择和多重比较伪影污染。

### H4：单方向不足，多维结构更接近事实

如果单一 refusal direction 只能解释 ALR，而不能解释 ASR 或 CompSP 错排，那么应该转向 harmfulness/refusal 双轴和 top-k safety subspace。近期文献也更支持“拒绝行为有共享控制旋钮，但具体拒绝类型/攻击风格由多个方向调制”。

## 4. 实验路线

### 阶段 A：复现和校准白盒方向

目标：把白盒侧的基础测量做稳。

任务：

- 用 harmful/benign anchor 构造层级 refusal/safety direction。
- 对方向符号做训练题校准，测试题上只评估，不再挑符号。
- 分别报告 ALR 和 ASR。
- 加入随机方向、词法 baseline、长度 baseline。

当前状态：已有第一次 Llama 初跑。结果显示简单方向对 ALR 很强，但未校准符号下的 wrong-agreement 有伪影。

### 阶段 B：有害性与拒绝分离

目标：判断 CompSP 更像跟踪 harmfulness、refusal，还是两者的组合。

设计：

- 复现 Zhao et al. 的 harmfulness/refusal 双方向思想。
- 在 instruction end token、assistant start token 或 answer prefix token 上分别提取分数。
- 做二维图：x=有害性投影，y=拒绝投影，颜色=ASR/ALR，描边=CompSP 排名桶。
- 检查高 ASR 低 CompSP、低 ASR 高 CompSP 是否落在可解释象限。

### 阶段 C：多维安全子空间

目标：从“一个方向”升级到“小维度结构”。

设计：

- 对 harmful/benign activation shift 做 PCA/SVD。
- 或按 Pan et al. 的正交安全方向思路，分析 top-k 方向。
- 对每个 prompt 生成 top-k 投影、子空间范数、到 harmful/benign center 的距离。
- 用这些特征解释 CompSP BTL 和错排案例。

### 阶段 D：跨攻击方案和跨模型边界

主证据只能使用目标模型一致的数据：

- Llama 主实验：`jbb-llama-pair`、`jbb-llama-drattack`，后续若补齐排名再加 `jbb-llama-ofa`。
- Qwen/4omini 只能作为跨模型对照，不能作为 Llama 标签。

对每个攻击方案单独报告，不直接混合。OFA、PAIR、DrAttack 的 prompt 分布不同，可能对应不同安全结构维度。

## 5. 统计与验收标准

核心指标：

- Spearman(`CompSP_BTL`, `ASR/ALR`)
- Spearman(`WhiteBox`, `ASR/ALR`)
- Spearman(`CompSP_BTL`, `WhiteBox`)
- partial correlation：控制 ASR/ALR 后的 CompSP-WhiteBox 相关
- two-way accuracy：同题内 pairwise 准确率
- wrong-agreement：`P(whitebox_label == compsp_label | compsp_label != empirical_label)`
- permutation baseline：同题内打乱 whitebox 分数或 CompSP 分数

最低验收：

- 层、符号、超参必须在训练问题上选择，在测试问题上固定。
- 每个结果都报告题目数、prompt 数、pair 数、baseline、置信区间或置换 p 值。
- 不把未校准 sign 的高 wrong-agreement 当主证据。
- 不用 Qwen 结果解释 Llama 标签。

## 6. 预期论文叙事

稳妥叙事：

> 黑盒安全代理可以学习到与目标模型安全行为相关的低维排序信号。这个信号不等同于经验 ASR，但与白盒模型中的拒绝/有害性结构存在部分对应。黑盒交互因此可能泄露关于模型安全边界的结构化信息。

更强叙事需要满足：

- 在留出问题上，CompSP 与白盒结构相关显著强于置换 baseline。
- 错排同向在符号校准后仍然存在。
- 双方向或多维子空间能解释单方向无法解释的案例。

如果强证据不足，退一步的论文也成立：

> CompSP 的排序误差揭示了经验 ASR、拒绝行为、有害性表征之间的错位。这个错位本身说明安全评估不能只看攻击成功率，还要看模型内部或代理可学习的安全结构。

## 7. 当前最大风险

- ASR 20 次采样噪声较大，ALR 可能更稳定但论文安全意义需要讲清楚。
- BTL 会放大局部 pairwise 误差，排序不准不一定等同模型理解错误。
- Llama 白盒方向只能解释 Llama 数据，跨模型结论要非常谨慎。
- 安全方向可能强烈受 prompt 长度、模板、语言、攻击格式影响。
- 多层多方向搜索容易产生多重比较伪阳性，必须有固定开发/测试划分。

## 2026-07-08 补充：伪安全结构可学习性实验

训练集固定为 `0~79` 中排除旧论文 20 个测试题后的 60 个问题；旧 CompSP LoRA 和旧 BTL 分数与目标模型、攻击方案、训练指标绑定，不能复用。本实验的新训练目标是白盒最后层 Ridge 伪安全方向投影的 pairwise 序关系，主证据是新训练 CompSP 在留出测试题上的 pairwise accuracy，BTL 只是可选后续分析。详见 `docs/pseudo_safety_learnability.md`。
