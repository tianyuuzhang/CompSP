# 安全结构研究讨论文档

这个文档用于记录我们接下来围绕“黑盒 CompSP 与白盒安全结构”的开放讨论。它不是最终论文，不要求一次写对；重点是把假设、反例、实验结果和你的判断持续沉淀下来。

## 当前共识

1. 前期 CompSP 论文是基础，不是后续研究的全部方向。
2. 新方向要解释“为什么黑盒排序可以泛化”，而不是只追求更高 ASR 预测准确率。
3. CompSP 的全局排序不准可能不是失败，而是一个窗口：它可能学到了比经验 ASR 更稳定的安全结构。
4. Llama 数据只能用 Llama 标签解释，不能拿 Qwen 的 ASR/ALR 当 Llama 主实验标签。
5. ALR 很可能比 ASR 更直接对应拒绝/安全方向，但 ASR 才是攻击成功的最终指标；二者都要保留。

## 我们需要一起判断的问题

### 1. 新论文的中心命题应该多强？

候选 A：黑盒交互可以蒸馏目标模型安全逻辑。

候选 B：黑盒代理学到的排序信号与白盒安全结构部分对齐。

候选 C：经验 ASR、拒绝行为、有害性表征之间存在系统错位，CompSP 错排揭示了这种错位。

我目前倾向 B 或 C。A 最有冲击力，但需要更强的跨设置证据。

### 2. “错得一致”到底算什么证据？

必须避免一种伪证据：白盒方向符号没校准，导致它和 ASR 反相关，于是 CompSP 出错时看起来与白盒同错。更公平的定义应该是：

- 在训练题上固定方向符号和层；
- 在测试题上计算 wrong-agreement；
- 与同题内置换 baseline 比较；
- 同时报告白盒对 ASR/ALR 的直接准确率。

### 3. CompSP 更可能学到 harmfulness 还是 refusal？

可能情形：

- 如果 CompSP 与 ALR/拒绝方向更相关，它学到的是“触发拒绝/长回答”的行为边界。
- 如果 CompSP 与 harmfulness 方向更相关，但与 ASR 不完全一致，它可能学到的是输入危险性结构。
- 如果它与二者组合相关，说明攻击成功需要同时降低拒绝信号、保留有害语义。

### 4. OFA 在新研究中的角色

OFA 数据很关键，因为它产生同题下大量 prompt 族，适合看排序结构。但目前 CompSP-白盒对齐主实验更依赖已有 BTL 排名，因此先用 Pair/DrAttack Llama 数据。OFA 可以作为：

- 白盒方向与真实 ASR/ALR 的大规模测试；
- 后续补齐 BTL 排名后的主实验；
- 与旧论文衔接的桥。

### 5. 反例如何处理？

如果 CompSP 与白盒方向对齐不强，不应硬解释。可以转向：

- CompSP 学到的是目标黑盒模型特有结构，而非 Llama 白盒结构；
- 单方向不足，需要多维安全子空间；
- ASR/ALR 标签本身噪声高，需要更稳定的行为指标；
- prompt 风格、长度、模板特征主导排序，需加入控制。

## 2026-07-08 补充：伪安全结构可学习性实验

训练集固定为 `0~79` 中排除旧论文 20 个测试题后的 60 个问题；旧 CompSP LoRA 和旧 BTL 分数与目标模型、攻击方案、训练指标绑定，不能复用。本实验的新训练目标是白盒最后层 Ridge 伪安全方向投影的 pairwise 序关系，主证据是新训练 CompSP 在留出测试题上的 pairwise accuracy，BTL 只是可选后续分析。详见 `docs/pseudo_safety_learnability.md`。

## 2026-07-11：回答通道探索账本

### 已观察到的事实

1. 单条回答文本比 q1 词法更能预测 Llama 的 ALR 伪方向和 ASR；同问题内打乱回答归属后，序准确率回到约 0.5。
2. 白盒回答最后层 embedding 没有超过黑盒 TF-IDF，至少在当前单层、固定 alpha 设置下，文本表面信号已经足够强。
3. 回答 TF-IDF 在 JBB-OFA 上跨模型迁移：ASR 大多在 0.59~0.66，ALR 在 0.72~0.86。
4. `ALR * judge_hazard` 提高独立后缀 ASR 的排序，却恶化 Brier；它更像风险排序器而非概率估计器。

### 候选解释

**H-R1：输出是安全状态的观测通道。**

模型面对同一 q1 时，输出中的拒绝、替代建议、具体性、语气和长度共同反映潜在安全状态。打乱回答归属使信号消失，支持这种解释。

**H-R2：跨模型迁移主要来自共享表层拒绝模板。**

ALR 的迁移显著强于 ASR，提示跨模型信号可能是“会不会长答/拒绝”的共同语言习惯，而不一定是共享内部安全方向。

**H-R3：加权 hazard 捕获判定边界附近的排序细节。**

首次 unsafe 的位置和有效票数把原 20 档 ASR 分解为更细的判定强度；这有利于排序，却因提前停止和条件化在概率校准上产生偏差。

**H-R4：ASR 与 ALR 对应不同结构层。**

ALR 易受拒绝模板和长度调制，ASR 还受攻击具体性、判定器误差和随机采样影响。若成立，白盒方向或黑盒输出特征不应期待对两者同样有效。

### 可证伪的下一步

1. 对跨模型矩阵加入 q1-only 基线。若回答相对 q1 没有增益，H-R1 不成立或主要是输入模板泄漏。
2. 删除高频拒绝短语、长度、列表格式后重跑跨模型矩阵。若迁移坍塌，H-R2 更可信；若仍保留，才考虑语义级共享结构。
3. 用重新完整三票判定的抽样回答验证 hazard。若排序增益消失，H-R3 只是旧提前停止流程伪影。
4. 用独立后缀 ASR、不同温度或后续新采样验证少回答文本模型。若预测只在同一 20 次样本内成立，不能称为利用潜在风险。
5. 比较输入 q1 embedding、回答 embedding 和输入+回答 embedding 的增量 R2/同题序准确率；若回答不增加，白盒机制解释应回到输入结构。
