## 审计结论

整理后的主流程与前期论文的实验逻辑基本一致：

1. OFA 将原始危险问题构造成 `q0`，由辅助模型生成大纲 `a0`，再拼接成攻击指令 `q1`。
2. 目标模型对每条 `q1` 重复回答，结果写入 `Ans`。
3. 长回答数量写入 `txl`，对应 `ALR * totQA`；unsafe 回答数量写入 `tx`，对应 `ASR * totQA`。
4. CompSP 训练/评估使用同一 `questionID` 内的指令对，默认过滤 `abs(delta) <= 0.15` 的噪声样本。
5. 推理阶段输出 pairwise 优势矩阵，再用 Bradley-Terry 拟合全局排序。
6. 白盒对齐实验只使用 Llama 标签数据，例如 `jbb-llama-pair` 与 `jbb-llama-drattack`；Qwen 结果只可作为跨模型对照，不能作为 Llama 标签。

## 已修复问题

### CompSP 推理 tokenizer 回退

问题：`src/compsp/ranking/evaluator.py` 原来直接从 LoRA 路径加载 tokenizer。实际检查 `100_pair_llama` 的 LoRA 路径时，该路径没有完整 tokenizer 文件，直接加载会失败。

旧代码 `/remote-home/zty/src/q02rank/evaluator.py` 的行为是：优先从 LoRA 路径加载 tokenizer，失败后回退到 base model 路径。

进一步检查发现，旧 LoRA 的 `adapter_model.safetensors` 中保存了 `embed_tokens.weight`，其行数为 `128257`；base tokenizer 长度为 `128256`。因此整理版不能简单按 `len(tokenizer)` resize。

修复：整理版 `PairwiseEvaluator` 已恢复 tokenizer 回退逻辑，并会优先读取 adapter 中的 embedding 行数来 resize base model。这样 `run_pairwise_matrix.py` 可以兼容旧实验中只保存 LoRA 权重、不保存 tokenizer 的目录。

## 逻辑核查结果

### OFA 流程

`src/compsp/ofa_pipeline.py` 的阶段划分为 `init`、`q1`、`answers`、`judge`、`all`，与论文中的 `q0 -> a0/q1 -> target answers -> txfilled` 一致。

需要注意：`judge_answers()` 当前使用长度阈值先统计 `txl`，然后只对长回答调用判定模型。这与论文附录中先用 ALR 长度启发式、再判定 dangerous response 的流程一致。

### 数据接口

`src/compsp/asr_dataset.py` 将 `tx/totQA` 定义为 ASR，将 `txl/totQA` 定义为 ALR。`Pair/DrAttack` 数据允许缺少 `q0/a0`，OFA 数据严格要求 `q0/a0`。

`questionID` 支持整数和 `Q_0` 形式字符串，加载后统一转为整数。这能兼容 Pair 数据中的字符串 `questionID`。

### Pairwise 样本

`make_pairwise_examples()` 只在同一 `questionID` 内构造组合，并默认使用 `min_delta=0.15`。这对应论文附录中丢弃 `|ASR_i-ASR_j| <= 0.15` 的噪声控制规则。

### BTL 排序

整理版 `src/compsp/ranking/ranker.py` 与旧 `q02rank/ranker.py` 的 Bradley-Terry 梯度更新逻辑一致。输出中的 `item_id` 从 `start_id` 开始连续编号，和 `data/saverk` 的旧输出格式兼容。

`PairwiseEvaluator.matrix()` 使用闭区间 `start:end+1`。旧 `q02rank/config.py` 中 `0_24` 对应 `size=25`，因此这里的闭区间规则是正确的。

### 白盒对齐

`scripts/run_whitebox_llama_alignment.sh` 当前只跑：

- `jbb-llama-pair`
- `jbb-llama-drattack`

对应的 CompSP 排名目录分别是：

- `data/saverk/save_jbb2100_pair_llama`
- `data/saverk/save_jbb4100_dr_ins_llama`

对齐脚本使用 `item_index` 匹配 `ranking_scores.json` 中的 `item_id`。这符合 Pair 数据中 `round` 不可靠的事实。

## 是否需要重跑实验

当前发现的真实逻辑问题只影响“未来使用整理版 `run_pairwise_matrix.py` 重新生成 pairwise 矩阵”的场景；已有 `data/saverk` 排名是旧 `q02rank` 生成的，不受这个整理版 bug 影响。

因此，本次不需要重跑已有白盒实验或 `data/saverk` 排名。修复后若未来重新跑 CompSP 矩阵，应使用新的整理版脚本。
