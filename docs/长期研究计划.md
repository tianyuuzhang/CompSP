# 长期研究计划

本计划将前期 CompSP 论文整理与后续“安全结构”研究分开，避免把旧论文方向误当成新论文主线。

## 阶段一：前期 CompSP 工程收口

目标：让现有代码、数据接口、说明文档可以稳定复现前期论文的关键实验。

任务：

- 保持 `/remote-home/zty/src` 和原始数据只读，只在 `tidy_CompSP` 内维护清洁版代码。
- 继续完善 OFA、ASR/ALR 数据接口、pairwise 样本构造、BTL 排序和 `data/saverk` 索引。
- 所有 README、docstring、实验报告默认中文；实验 prompt 保持原文。
- 对每个新增脚本，在同目录文档中写清输入、输出、调用方式和论文逻辑对应关系。

验收：

- `validate_asr_data.py` 能抽样或完整检查注册数据。
- `run_pairwise_matrix.py` 能兼容旧 LoRA 目录。
- `run_bt_rank.py` 输出与旧 `q02rank` 格式兼容。
- 文档明确区分 Llama/Qwen/GPT-4o-mini 标签，避免跨模型误用。

## 阶段二：安全结构白盒复现

目标：复现并扩展白盒安全方向/安全子空间结果，用 Llama3.1-8B 作为可控模型。

任务：

- 复现单方向 harmful-minus-benign / refusal direction。
- 分离有害性方向和拒绝方向，比较它们对 ASR 与 ALR 的预测力。
- 做多维方向：PCA/SVD、安全子空间、正交方向。
- 加入 sign 校准、留出问题评估和 shuffle baseline。

验收：

- 每个方向实验都有固定数据划分、输出路径、日志和中文结果报告。
- 不使用 Qwen 数据作为 Llama 标签。
- 报告中区分“相关性强”“可解释结构一致”“错因一致”三种不同强度的结论。

## 阶段三：黑盒代理与白盒结构对齐

目标：判断 CompSP 学到的黑盒排序信号是否与白盒安全结构存在一致性。

任务：

- 用 `jbb-llama-pair` 和 `jbb-llama-drattack` 做初步对齐。
- 若找到或生成 Llama OFA 的 BTL 排名，再加入 `jbb-llama-ofa`。
- 以 `item_index` 对齐白盒分数、CompSP BTL 分数和真实 ASR/ALR。
- 分析正确一致、错误一致、符号伪影和随机 baseline。

验收：

- 至少报告 ALR 与 ASR 两套结果。
- 每个结论都包含样本数、pair 数、准确率、相关系数和 baseline。
- 对“不支持假设”的结果也如实记录，避免后续论文建立在不稳定信号上。

## 阶段四：论文叙事收敛

目标：把前期黑盒结果和后续白盒安全结构整合成清晰叙事。

建议叙事：

- 前期 CompSP 说明黑盒交互中存在可学习的安全行为排序信号。
- 白盒方向说明 Llama 内部存在低维、连续的安全相关结构。
- 核心问题不是“CompSP 是否精确预测 ASR”，而是“黑盒代理学到的排序信号是否与白盒安全结构共享某种潜在维度”。
- 如果错因一致不强，也可以转向更稳妥的问题：哪些安全结构能被黑盒交互稳定拟合，哪些不能。
