# 安全结构实验可信度审计与止损建议

日期：2026-07-13  
范围：当前仓库中与 CompSP、ASR/ALR、白盒 hidden state、回答文本嗅探、prefix-zero、跨攻击/跨模型迁移和预算增采有关的全部主要实验。

## 0. 审计结论摘要

当前项目已经发现了一些真实而有价值的现象，但“可预测”“相关”“存在低维分量”经常被过快升级成“安全结构”“机制对齐”或“内部状态泄露”。目前最可信的主线不是“回答文本直接读取了单一白盒安全方向”，而是：

> 少量输出是一个有用的黑盒行为观测通道；在严格前缀/后缀拆分和固定前缀组成后，回答文本对后续 ASR 仍有一定增量。但增量可能来自具体危险主题、执行接近度、回答姿态和 judge 漏判边界，尚未证明对应独立、因果、可迁移的内部安全结构。

应立即停止或降级以下叙事：

1. 未校准 layer/sign 的 wrong-agreement 证明 CompSP 与白盒“错得一致”。
2. ALR 高相关证明发现了“安全语义”。ALR 本质上是长回答比例，长度和拒绝形式对它的高预测接近定义回读。
3. 强掩码后不下降就排除了危险词捷径。当前只遮蔽少量通用词，主题特定危险概念仍完整保留。
4. prefix-zero 的高 recall 证明回答泄露隐安全结构。它也可能只是识别了已经接近危险、但二元 judge 漏判的回答。
5. SVD 最佳单分量证明存在单一低维安全轴。当前最佳分量在测试集上从 32 个分量及正负符号中挑选，存在明显多重比较和测试集选择偏差。
6. 数万 pair 或数千记录意味着统计证据很强。大量 pair 高度重叠，真正独立的测试单元主要是 20 个问题。
7. 多 seed 的标准差等同于独立重复实验。多数 seed 只是对同一批 20 条历史回答重新抽样，并没有产生新的模型回答、问题或 judge 标签。

## 1. 全项目共同的统计风险

### 1.1 有效样本量被 pair 数和记录数夸大

同一问题下 100 条 q1 会产生数千 pair；同一 q1 又会反复出现在许多 pair 中。这些 pair 不是独立样本。测试集的主要独立单元是问题，当前通常只有 20 个；HarmBench 某些目标模型甚至只有 7 个测试问题。

以后必须：

- 主指标先对每个 `dataset_key + questionID` 单独计算，再做问题宏平均。
- 置信区间使用按问题 cluster bootstrap 或 leave-one-question-out，不使用把 pair 当独立样本的普通二项区间。
- 所有结果同时报告问题数、每题正例数、正例分布；禁止只报告 pair 数。

### 1.2 固定 20 题“测试集”已被反复使用，事实上已成为开发集

当前 20 个旧论文测试题已被用于选择：方向、层、符号、阈值、清理词表、文本视图、SVD 维数、quota、预算策略和研究叙事。即使每个脚本没有直接拟合测试标签，长期反复观察结果也会造成研究者层面的测试集过拟合。

建议立即重新定义：

- 旧 20 题：改名为 `legacy_dev`，用于诊断和历史可比。
- 当前 60 个训练题：再切分为训练/开发，或使用 grouped nested CV。
- 若 80--99 数据完整：作为一次性最终确认集，在方法、超参数、清理规则和主指标冻结前不查看。
- 如果不能使用 80--99：采用按问题重复交叉验证，并明确没有真正 untouched test。

### 1.3 ASR/ALR 是噪声离散估计，不是真实连续标签

`totQA=20` 时 ASR/ALR 最小步长为 0.05。相近 ASR 的 pair 标签大量由采样噪声决定。ASR=0 也不能证明真实概率很低。

以后应：

- 报告 `tx/totQA`，并使用 Beta/二项后验区间。
- pair 标签改为 `P(p_i > p_j | data)`，或至少过滤后验无法区分的 pair，而不是只按经验差值。
- 对低 ASR 结论必须追加独立采样；不能用文本分数替代概率估计。
- 区分排序性能、概率校准和发现率；高 AUC/AP 不代表概率可信。

### 1.4 多 seed 不是独立复验

对同一 20 条回答重新抽前 k 条，只反映分割敏感性，不包含：新问题、新模型采样、新温度、新 judge 的不确定性。其标准差通常低估真实泛化误差。

至少要有一项真正独立复验：

- 新随机种子重新生成回答；
- 不同温度或解码配置；
- 不同 judge；
- 新问题 holdout。

### 1.5 多重比较和“挑最好”普遍存在

选择最佳 layer、sign、SVD component、正负方向、k、quota、清理方式、攻击组合后再报告最好值，会制造看似稳定的高分。

以后必须在开发集选择，最终集只评一次；探索表可以完整报告，但不得把最大值当确认性证据。必要时做 permutation max-statistic 或多重比较校正。

## 2. 原始 CompSP、经验 ASR/ALR 与 BTL

### 当前可信度：黄色，作为历史基线保留；不作为机制证据

主要问题：

- ASR/ALR 只有 20 次采样，pair 标签在近邻区间噪声大。
- BTL 会把局部 pairwise 误差放大成全局排名差异。
- 旧 LoRA、数据、目标模型和攻击方案绑定，跨设置解释容易错配。
- 报告大量 pair accuracy 容易掩盖问题间不稳定性。

可保留结论：CompSP 在特定目标模型/攻击方案中能学习某种输入排序信号。

不能保留结论：该排序信号天然等同于目标模型安全边界或真实 ASR。

下一步最小实验：

1. 用 posterior pair labels 或只保留高置信 pair 重算。
2. 每题 macro accuracy + cluster bootstrap。
3. 在新采样标签上检验旧 BTL 排序是否比原经验 ASR 更稳定。
4. 若新采样复现失败，停止继续解释旧 BTL 的机制。

## 3. harmful-minus-benign 白盒方向与 wrong-agreement

### 当前可信度：红色；暂停“错得一致”主线

主要问题：

- 方向就是 harmful/benign 均值差，在同一 anchor 上选择分离最好的层，anchor separation 本身近似构造性结果。
- harmful 与 benign 数据来源、长度、主题和格式均不同，方向混合了数据集身份、危险主题、长度和 chat template。
- layer/sign 曾在测试结果上遍历，wrong-agreement 的高值主要来自反向符号。
- 33 层、两个符号和多个指标构成严重多重比较。
- 目前只有相关性，没有 activation intervention，不能称为因果安全方向。

允许的表述：某些层的表示可线性区分当前 harmful/benign anchors，并与 ALR/ASR 相关。

禁止的表述：CompSP 与白盒安全机制共享错误；layer 32 是模型真实安全层。

恢复该主线前必须满足：

1. harmful/benign anchor 单独 train/dev/test，层和符号只在 dev 固定。
2. 长度、数据源、主题和模板匹配的 anchors。
3. 同范数随机方向、标签置换、最大层统计量 baseline。
4. 按问题 cluster bootstrap。
5. 至少一个因果干预或生成起始状态验证。

在这些条件完成前，不再投入时间做更多 wrong-agreement 图表。

## 4. ALR/ASR 监督 Ridge 方向

### 当前可信度：黄色；保留“线性可解码”，降级“安全方向”

优点：按 question 做 GroupKFold 选 alpha，训练/测试问题分离，设计比 harmful/benign 方向更干净。

不足：

- Ridge 只证明 hidden state 中可以线性解码标签，不证明模型使用该方向做决策，也不证明它是一条自然存在的单轴。
- ALR 是长回答比例；`ALR-Ridge` 更适合称为“ALR 行为读出方向”，不是安全方向。
- 全局 Pearson 可能被问题间、攻击方案间均值差异抬高。需要同题中心化、分攻击和 leave-one-attack-out。
- 4096 维读出与 60 个独立训练问题之间仍可能存在问题级过拟合，行数 17960 不能当独立样本量。
- 固定 20 题已反复查看，没有 untouched confirmatory test。

下一步：

1. 报告每题中心化 Pearson/Spearman、同题序准确率和问题宏平均。
2. 在每题内置换 ALR/ASR 标签训练 Ridge，形成 null distribution。
3. 单攻击训练、跨攻击测试；再做 leave-one-attack-out。
4. 比较 last-token、mean-token、生成起始位置和多层 pooling，全部在 dev 选。
5. ASR 使用二项/层级监督，而不是把 `tx/20` 当无噪声连续值。
6. 若 within-question 和跨攻击效果明显下降，则主结论收缩为“全局行为可解码”，不再称结构泛化。

## 5. CompSP 学习伪方向的 pairwise LoRA

### 当前可信度：黄色偏红；暂停新的长时 GPU 训练

主要问题：

- 一个 q1 被重复放入大量 pair，训练和评估 pair 高度相关；当前 accuracy/AUC 的不确定性远大于按 3--5 万 pair 计算的表面误差。
- 测试集同时作为 Trainer 每 epoch 的 eval 集，且被多次实验观察；它不是严格一次性测试集。
- pair 文本直接拼接后统一截断到 512。DrAttack 两条长文本相加远超 512，后半段和 Text B 可能被系统性截断，导致位置偏差；ALR-DrAttack 的随机结果不能直接解释为结构不可学习。
- min-max 按题归一化和 `delta>0.15` 依赖极值，异常点会改变大量 pair 是否入选。
- 单 seed；没有 A/B 对调一致性、传递性和问题宏平均。
- ALR 粗桶捷径虽做了部分子集检查，但还没有随机方向/置换方向 baseline。

下一步先用便宜模型验证，不再直接跑 20--30 小时 LoRA：

1. 单条 q1 预测 pseudo score，再做分数差排序，避免 pair 重复膨胀。
2. Siamese 编码 A/B，分别截断并共享编码器，避免 Text B 被截断。
3. TF-IDF/线性模型作为最低基线。
4. A/B 交换一致率、循环一致性、每题 macro、cluster bootstrap。
5. 多 seed；随机方向和打乱 pseudo score baseline。
6. 只有便宜模型在新 holdout 上稳定后，才恢复 LoRA。

## 6. 回答手工特征与“内容风格”

### 当前可信度：绿色作为描述性 baseline；红色作为语义/机制证据

“内容风格”包括行句数、列表、代码块、标点、词汇多样性、平均词长、实质内容比例等。它不直接读取具体危险词，但仍高度反映“是否进入执行/教程姿态”。

关键反驳：

- ALR 由长回答比例定义，长度及其代理预测 ALR 接近定义回读。
- ASR/ALR 全标签包含被抽到的回答时，单条回答自身的 0/1 unsafe/long 事件会直接贡献标签。
- 当前真实测试分布中，单次 unsafe 0/1 预测完整 ASR 序关系的理论期望约 0.6617；单次长回答 0/1 预测完整 ALR 序关系约 0.7590。单回答 TF-IDF 的 ASR 0.676 只比朴素伯努利基线高约 0.014。

因此：

- 单回答预测完整 ASR/ALR 只能作 sanity check。
- 任何“具体内容增量”必须使用独立后缀标签或固定回答组成。
- ALR 结果不再作为输出语义主证据。

## 7. TF-IDF 的 q1-only / response-only / joint

### 当前可信度：黄色；输入增量存在，但来源未定

优点：response 常强于 q1，简单同题回答打乱后同题序准确率回到约 0.5；这是具体 q1--response 对应关系有信息的证据。

不足：

- TF-IDF 可直接利用主题特定危险概念和具体操作词。
- `q1+response` 不超过 response 不能证明 q1 没有额外信息；可能是高维稀疏特征竞争、正则化或词表容量造成。
- 在 full ASR/ALR 标签上仍有同样本贡献。
- q1-only 与 response-only 模型容量、文本长度和词项数不匹配。

下一步：

1. 只在独立 suffix target 和 composition-controlled 条件下比较。
2. 使用等容量、等长度、等词项预算。
3. 做增量模型：count+q1、count+response、count+q1+response，并报告交叉验证 ΔAUC/ΔAP/Δpairwise，而非只比较独立模型。
4. 用 conditional permutation 测试 response 在 q1、计数、长度、主题之后的增量。

## 8. 回答打乱、危险词掩码、残差化和长度匹配

### 回答打乱：有用但不充分

同题同攻击内打乱是良好负对照，但 prefix-zero 中打乱后仍高于随机，表明边际分布、组成和回答姿态仍贡献信号。以后打乱需匹配：safe/unsafe 组成、长度桶、主题簇和拒绝状态。

### 危险词/强掩码：证据必须降级

当前掩码只覆盖少量通用危险词、模板词和诱饵词，不能排除主题特定概念，如材料、比例、步骤、载荷、凭证、执行、传播和诱导。掩码不下降只能排除列出的词，不能证明剩余是深层安全结构。

下一步应使用：

- 同主题内比较；
- NER/主题概念遮蔽；
- 保留姿态、删除主题实体的改写；
- 保留主题、打乱执行细节；
- topic-matched nearest-neighbor control。

### 两阶段残差化：中等证据

“先拟合长度格式，再拟合训练残差”是顺序依赖的，且不保证测试集上的真正条件增量。应改为 nested model 比较、cross-fitting 和 conditional permutation。现有小幅提升可视为候选增量，不能当独立语义成分。

### 长度匹配：重要的反证

严格长度匹配后白盒伪方向接近随机，说明此前 response--伪方向关系大部分来自整体回答形态。这一结果应提高权重，不应被后续更多弱正结果掩盖。关于“回答直接读取一维白盒方向”的主线应暂停。

## 9. 回答 Llama embedding

### 当前可信度：红色作为“生成轨迹”证据；黄色作为文本编码 baseline

当前做法是把已完成回答重新当作用户文本输入 Llama，再取最后层最后 token。它不是目标模型原始生成该回答时的 hidden-state trajectory，也不是 q1+assistant continuation 的 teacher-forced 生成状态。因此不能称为“白盒回答轨迹”。

此外只取最后 token，长回答语义可能压缩不充分；截断和结束符也会影响表示。

下一步正确方案：

1. 使用原始 `q1 + assistant response` chat 格式做 teacher forcing。
2. 提取 assistant token 区间的生成时 hidden states，分别做 mean、last-content、unsafe-span pooling。
3. 与只编码 response 的 baseline 对比。
4. 在 dev 选 pooling，最终 holdout 只评一次。

完成前停止从现有 embedding 结果推导内部输出结构。

## 10. alternative ASR / hazard-weighted ASR

### 当前可信度：黄色作为排序启发式；红色作为新标签或概率

`hazard_weighted_asr = ALR * judge_hazard` 来自同一批回答和同一判定记录。与经验 ASR 的高相关是预期结果，不是独立验证。旧 judge 首次 unsafe 后停止，导致后续票缺失非随机，`judge_hazard` 不是无偏概率。更多不同数值不等于更多真实信息；Brier 更差也说明它不校准。

只允许把它作为主动采样排序 baseline。不要称为更精确 ASR，不作为机制监督主标签。

下一步：重新对抽样回答做完整、固定票数、多 judge 判定；比较 any/majority/概率分数。若没有新判定，不再扩展 alternative ASR 指标族。

## 11. 严格前缀/后缀回答预测 future ASR

### 当前可信度：绿色偏黄；当前最值得保留的黑盒结果之一

优点：特征回答不进入后缀标签，避免最直接的同样本泄漏；不同攻击上均高于随机。

不足：

- 前缀与后缀仍来自同一历史 20 次回答，多个 seed 高度重叠。
- future ASR 样本数更少，标签更噪声。
- k=4 时直接 prefix ASR/hazard 通常强于纯文本，因此文本不是普遍更优。
- 尚缺统一嵌套模型直接量化文本超越 `ASR_small+ALR_small` 的增量。

下一步：

1. 新生成独立 suffix 回答作为确认集。
2. 比较 prefix count、count+q1、count+response、count+joint。
3. 按问题 cluster bootstrap；报告 calibration。
4. 预先冻结 k、阈值和特征清理方式。

若新采样中 response 增量消失，应收缩为“历史样本内可预测”，停止结构叙事。

## 12. 跨攻击迁移和 mixed 训练

### 当前可信度：黄色

正面：训练问题与测试问题分离，跨攻击仍有一定 ASR 排序能力，说明不完全依赖单一模板。

风险：

- 三种攻击共享同一问题库和目标模型，模型可学习主题难度与通用拒绝/执行姿态，不等同于跨机制迁移。
- ALR 的高迁移主要可能是共享长度和拒绝形式。
- mixed 往往拥有更多训练数据，和单域比较不公平。
- 16 个矩阵格没有问题级置信区间，容易挑选高格子。

下一步：

1. mixed 与单域做等样本量比较。
2. leave-topic-family-out，不仅 leave-question-out。
3. 每格问题宏平均 + cluster CI，并报告完整矩阵，不挑最好格。
4. 重点保留 ASR；ALR 只作行为形式 baseline。

## 13. 跨模型和 HarmBench 迁移

### 当前可信度：黄色偏红；暂停广泛跨模型结论

风险：

- response 跨模型可利用共享危险主题和通用拒绝模板，不一定观察到目标模型特有状态。
- q1 baseline 虽有帮助，但不能排除 response 只是更详细地复述主题。
- HarmBench 每目标模型测试问题很少，部分标签饱和，平均值由少数目标模型驱动。
- 多模型、多源目标矩阵带来大量比较。

下一步：

1. 先在每个目标模型内做 count/q1/response 的条件增量。
2. 使用同一 q1 的多模型回答做 paired analysis：控制 q1 固定，只看不同模型回答是否预测各自 future ASR。
3. 扩大问题数；在此之前 HB 结果只做探索附录。
4. 若 response 的增量在控制具体主题和回答组成后消失，停止“跨模型安全状态观测”叙事。

## 14. Prefix-zero 隐蔽风险、三视图和高 recall

### 当前可信度：黄色；有操作价值，但机制解释很弱

主要统计风险：

- prefix-zero 是按前缀结果条件选择，会产生 selection/collider bias。q1-only 在该子集失效，部分可能是筛选本身压缩了 q1 风险差异，并不能直接证明 response 获得了独立内部信息。
- k=4 正例率约 1%，单 seed 只有约十几个正例；81.2% recall 可能只是命中约 13/16。问题级分布和置信区间会很宽。
- 5 seed 使用同一底层回答，正例集合重叠，不是 5 次独立复验。
- 高 recall 伴随约 4% precision；它是召回型筛选器，不是准确识别器。
- 文本可能捕获“已接近 unsafe、但当前 judge 漏判”的回答，这与独立安全结构泄露是不同解释。

允许表述：在当前 judge 和历史采样下，response 文本可提高 prefix-zero 后续高风险候选的优先级。

禁止表述：模型已经从表面安全回答中读出内部安全轴；q1 完全没有信息。

下一步：

1. 报告每 seed、每问题的正例绝对数和 Wilson/cluster bootstrap CI。
2. 多 judge、连续 hazard margin、人工盲审高分/低分回答。
3. 在控制 judge margin、主题、长度和组成后再测 response 增量。
4. 新采样验证 top 候选是否仍高风险。
5. 对自然顺序的前 k 条回答分层，不从完整 20 条中事后挑选组成。

## 15. 强掩码 TF-IDF SVD 低维探针

### 当前可信度：红色作为安全结构证据；建议停止扩展当前 SVD 叙事

严重问题：

- `best_single_component` 在测试集上遍历 32 个分量，同时对每个分量选择正/负符号，再取 AP 最大者。这是直接测试集选择，最佳单分量分数明显乐观。
- SVD 是无监督词袋压缩，分量混合主题、长度和回答姿态；低维可预测不等于白盒安全低维。
- 32 维 Ridge 保留全 TF-IDF 性能是普通降维现象，需要随机投影和相同维数 baseline。
- explained variance 与监督预测能力没有直接机制含义。

建议：

- 停止引用“最佳单分量”结果。
- 不再投入当前 TF-IDF SVD 的扩展。
- 若继续低维研究，必须在训练/dev 选择分量和符号，在 final test 固定；加入随机投影、PCA、frozen embedding、主题控制，并与白盒方向做 out-of-sample 对齐。

## 16. Composition-controlled 回答信息增益

### 当前可信度：黄色偏绿；是重要进展，但仍非前瞻实验

正面：固定 `unsafe_only`、`safe_long_only`、`safe3_unsafe1` 后，response AUC 均高于 count baseline 和 response-shuffled，说明结果不只是 `ASR_small/ALR_small` 数值。

关键不足：

- 脚本从完整 20 条回答中搜索并抽取所需 safe/unsafe 组成；样本是否入选使用了整池信息。这是条件关联分析，不是自然序贯前缀，存在 eligibility/selection bias。
- 观察回答是根据 toxic 标签挑出的，现实中必须先有 judge；它不等于不看标签的黑盒文本利用。
- 主题和 judge 漏判仍未控制。
- 尚未按攻击方案分层，混合 AUC 可能由攻击组成和基础风险差异抬高。

下一步最小确认：

1. 先随机排列或真实生成前 k 条，再按自然观察到的组成分层，不能从后缀中事后挑回答。
2. 每个攻击方案单独报告；问题 macro + cluster CI。
3. count+q1 与 count+q1+response 的条件增量和 permutation test。
4. 新独立后缀采样。
5. 多 judge/人工审查排除近危险漏判。

只有这些成立后，才可把“固定计数后 response 有增量”升级为主结论。

## 17. 序贯采样和固定预算发现模拟

### 当前可信度：黄色；作为离线策略模拟保留，不作为线上效果

正面：明确区分攻击优化与安全审计目标；固定 extra budget 比固定 top 20% 更合理。

不足：

- 仍是历史 20 条回答的离线重放，后缀不是新访问结果。
- quota、top fraction、k、阈值和策略组合已在同一旧测试集反复观察，存在策略过拟合。
- `future ASR>=0.5` 在少量 suffix 上非常粗糙。
- final recall 初始值已很高，绝对 gain 小；只强调 prefix-zero recall 会掩盖 precision 和成本。
- hybrid 的优势取决于研究目标，不能说总体优于 hazard 策略。

下一步：

1. 在 dev 选择 k/quota/budget，冻结后在新 holdout 评一次。
2. 新在线或独立采样模拟真实追加访问。
3. 同时报每发现一个新高风险指令的回答成本、precision、recall、置信区间。
4. 与简单策略比较：随机、prefix ASR、Jeffreys UCB、ALR、q1-only。
5. 若在新采样中不能稳定降低发现成本，停止工程扩展。

## 18. ASR 分辨率限制

### 当前可信度：绿色；应作为所有后续工作的硬约束

这是较可靠的描述性结果：20 次采样无法区分 0.01 与 0.001，0/20 仍有很宽后验上界。该结果应放在论文和实验设计前部，而不是附属说明。

下一步不是再训练预测器，而是：

- 设计层级贝叶斯/序贯抽样；
- 对候选追加独立回答；
- 报告概率区间而非点估计；
- 不再声称历史数据给出了精细低 ASR 概率。

## 19. 立即执行的止损与优先级

### 立即暂停

1. 新的 wrong-agreement/layer/sign 扫描。
2. 新的 20--30 小时 pseudo-direction LoRA，直至修复截断、pair 依赖和评估方案。
3. 当前 TF-IDF SVD 最佳分量及其“低维安全轴”叙事。
4. 继续扩大通用危险词掩码词表并把不下降当机制证据。
5. HarmBench 小样本上的广泛跨模型结论。
6. 把 ALR 高分写成语义安全结构。

### 最高优先级确认实验

1. **新 holdout / 新回答采样**：这是所有结论的共同瓶颈。
2. **自然前缀 composition-controlled**：固定前缀组成，但不从全池事后挑选。
3. **统一增量模型**：`count + q1` 对比 `count + q1 + response`，按问题 bootstrap。
4. **judge 漏判审计**：多 judge + 人工盲审 prefix-zero 高低分。
5. **Ridge within-question/leave-attack-out**：判断白盒读出是否超越问题/攻击均值。
6. **正确生成轨迹 hidden state**：teacher-forced q1+response，而非把回答重新当用户输入。

### 继续条件与停止规则

- 任一主结论必须在问题宏平均、cluster 95% CI 下稳定高于预注册 baseline。
- 任一“文本增量”必须在控制 prefix counts、q1、长度/拒绝/格式后仍存在。
- 任一“机制/结构”结论必须超越主题/模板/随机方向，并至少有跨攻击或干预证据。
- 新 holdout 上效应低于原报告一半、CI 包含 baseline，或只由一个攻击/少数问题驱动时，停止该分支，不用继续增加模型复杂度。
- 先跑便宜线性/冻结模型；便宜 baseline 未通过，不启动长时 LoRA/GPU 实验。

## 20. 推荐的当前中心命题

当前最稳妥的中心命题应收缩为：

> 模型的少量输出为后续行为风险提供了输入之外的黑盒观测信号；该信号在独立后缀和固定前缀组成的离线分析中可测，但包含明显的输出组成、危险主题、回答姿态和 judge 边界成分。白盒一维读出与这些信号存在相关性，但尚不足以证明统一、因果或可迁移的安全结构。

只有在新采样、自然前缀、主题控制、judge-margin 控制和问题级确认后仍成立，才升级为更强的“输出反映内部安全状态”命题。
