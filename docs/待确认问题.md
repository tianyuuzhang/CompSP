# 待确认问题

这些问题不会阻塞当前代码整理，但会影响后续论文叙事、实验范围或结果解释。

## 研究口径

1. 前期英文论文中的 CompSP 结果是否只作为“黑盒安全结构可学习”的证据，而不再作为后续论文主贡献？

   当前整理默认：是。后续主线应转向安全结构，CompSP 作为黑盒侧证据和实验工具。

2. 白盒方向实验中，是否把 ALR 作为主要结构指标、ASR 作为补充指标？

   当前结果显示 ALR 与简单 harmful-minus-benign 方向更稳定对齐。ASR 更接近最终危险性指标，但噪声更大。

3. “CompSP 和白盒方向错得一致”是否还作为主要假设？

   当前初跑不支持直接这样写。更稳妥的表述是：先校准白盒方向符号，再在留出问题上检查错误一致性，并加入 shuffle baseline。

## 数据缺口

1. `jbb-llama-ofa` 有原始 ASR/ALR 数据，但目前没有找到对应 OFA 的 `data/saverk` BTL 排名。

   如果要把 OFA 纳入白盒-CompSP 对齐，需要先为 Llama OFA 生成或定位对应排名。

2. Pair/DrAttack 数据不是所有问题都有完整 100 条记录。

   当前接口会跳过缺失文件，但若做严格论文表格，需要明确每个实验实际使用的问题数和样本数。

3. HarmBench 的部分 OFA 文件记录数不足 100。

   这些文件可用于探索，但正式统计最好单独列出缺失情况，避免默认分母错误。

## 实验解释风险

1. 白盒方向的 sign 必须先校准，否则高 wrong-agreement 可能只是符号反向造成的伪影。

2. `round` 在 Pair 数据中不可靠。所有排序和白盒对齐应使用 JSON 列表下标 `item_index`。

3. Qwen、GPT-4o-mini、Llama 之间 ASR/ALR 相关性只有中等，不能跨模型混用标签。

4. 英文 prompt 模板是否需要在论文中原样公开，需要你确认最终论文是否使用中文正文配英文 prompt 附录。
